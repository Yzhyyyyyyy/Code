# 概率与期望 DP：记忆化搜索 (DFS) 通用指南

在算法竞赛（如 CCF CSP、ACM 等）中，处理复杂的概率与期望 DP 时，直接写 `for` 循环递推往往面临**状态难以初始化**、**遍历顺序难以确定**、**无效状态过多**等痛点。

**记忆化搜索（DFS + 备忘录）** 是解决这类问题的终极武器。它的核心优势在于：**从起点出发，让递归栈自动帮我们寻找终点，并在回溯时完成推导。**

---

## 核心口诀：正推求概率，倒推求期望

虽然我们写代码时都是从起点 `dfs(起点)` 开始调用，但它们在数学定义和边界条件上有着本质的区别：

| 维度 | 求概率 (Probability) | 求期望 (Expected Value) |
| :--- | :--- | :--- |
| **状态定义 `dfs(S)`** | 从状态 $$S$$ 出发，最终**到达目标状态的概率**。 | 从状态 $$S$$ 出发，最终**到达目标状态所需的期望代价（步数）**。 |
| **终点边界条件** | 已经到达目标：`return 1.0` (必然到达)<br>进入死胡同：`return 0.0` (绝对无法到达) | 已经到达目标：`return 0.0` (不需要再走任何步数了) |
| **状态转移方程** | $$P(S) = \sum (p_i \times P(next_i))$$ | $$E(S) = \text{当前代价} + \sum (p_i \times E(next_i))$$ |
| **物理意义** | 概率的累加（乘法原理） | 步数/代价的累加，加上未来期望 |

---

## 场景一：求概率

**问题模型**：从初始状态出发，经过一系列随机选择，求最终成功达成某个目标的概率。

**DFS 视角**：`dfs(当前状态)` 询问的是：“我站在这里，未来有多大可能走到终点？”
- 如果我现在就站在终点，那可能性就是 100%（返回 1.0）。
- 如果我掉进了永远到不了终点的陷阱，可能性就是 0%（返回 0.0）。
- 否则，我的概率等于**我走向各个分支的概率 $$\times$$ 那个分支能走到终点的概率**之和。

### 💻 C++ 模板

```cpp
double dp[MAX_STATE];
bool vis[MAX_STATE];

double dfs_prob(int state) {
    // 1. 边界条件：到达目标状态
    if (is_target(state)) return 1.0;
    
    // 2. 边界条件：进入死胡同（无法到达目标）
    if (is_dead_end(state)) return 0.0;
    
    // 3. 记忆化：算过就直接返回
    if (vis[state]) return dp[state];
    
    double prob = 0.0;
    
    // 4. 遍历所有可能的下一步
    for (int i = 0; i < choices; i++) {
        int next_state = get_next(state, i);
        double p = get_prob(state, i); // 走到下一步的概率
        
        // 累加：走向下一步的概率 * 下一步到达终点的概率
        prob += p * dfs_prob(next_state); 
    }
    
    vis[state] = true;
    return dp[state] = prob;
}
```

## 场景二：求期望

**问题模型**：从初始状态出发，每次行动有不同概率转移到不同状态，求到达目标状态所需的平均行动次数（或总花费）。（如：收集卡牌问题）

**DFS 视角**：`dfs(当前状态)` 询问的是：“我站在这里，平均还要花多少代价才能走到终点？”
- 如果我现在就站在终点，那我不需要花任何代价了（返回 0.0）。
- 否则，我只要采取行动，就一定会产生基础代价（比如抽卡 1 次，代价为 1）。然后加上**走向各个分支的概率 $$\times$$ 那个分支还要花的代价**。

### 💻 C++ 模板

```cpp
double dp[MAX_STATE];
bool vis[MAX_STATE];

double dfs_expected(int state) {
    // 1. 边界条件：到达目标状态，不需要再花费代价
    if (is_target(state)) return 0.0;
    
    // 2. 记忆化：算过就直接返回
    if (vis[state]) return dp[state];
    
    // 3. 基础代价：只要还没到终点，当前这一步必然要走（比如抽卡1次）
    double expected_cost = 1.0; 
    
    // 4. 遍历所有可能的下一步
    for (int i = 0; i < choices; i++) {
        int next_state = get_next(state, i);
        double p = get_prob(state, i); // 走到下一步的概率
        
        // 累加：走向下一步的概率 * 下一步到达终点还需要的期望代价
        expected_cost += p * dfs_expected(next_state); 
    }
    
    vis[state] = true;
    return dp[state] = expected_cost;
}
```

## 🧠 进阶感悟：破解 DFS 记忆化搜索的底层逻辑

在真正编写和理解 DFS 期望 DP 时，牢记以下三大核心心法，就能彻底看透它的运行机制：

### 1. 顺着写，倒着算（触底反弹）
- **代码逻辑是顺流而下的**：我们写代码时是顺着人类直觉写的，即 `dfs(当前) -> dfs(未来)`。这极大地减轻了思维负担，不需要去烧脑思考状态的拓扑排序。
- **实际执行是倒着累加的**：当程序在 CPU 里运行时，它会一头扎到底，直到撞上边界条件（终点）。在撞到终点之前，所有状态都在“排队等结果”。
- **降维打击——都是已知数据**：一旦终点返回了确定的值，程序就开始**倒着向回累加**。当代码执行到 `res += p * dfs(未来)` 时，这个 `dfs(未来)` **绝对已经是一个算得清清楚楚的常数了**！因此，倒着累加没有任何逻辑漏洞，永远是用已知的未来推导当前的未知。

### 2. 向前看，绝不向后看
- DFS 的精髓在于**探索未来**。站在当前状态时，**绝对不要去管“上一次抽中了什么”或者“我是怎么来到这里的”**。
- 只需要关心：“我接下来可能遇到哪些情况？”然后把下一次的期望乘以概率加到自己身上即可。

### 3. 期望的“1”是局部代价，不是全局初始化
- 期望 DP 中的 `+1`（例如 `double expected_cost = 1.0;`），**不是**放在全局数组初始化里的。全局的 `dp` 数组初始状态是空的（未计算的）。
- 这个 `1` 是每次进入未计算过的状态时，只要还没到终点，当前这一步必然要付出的**基础代价**。它是随着 DFS 的深入，在每个有效状态上临时赋予并累加起来的。

---

## 💡 总结：为什么 DFS 这么好用？

1. **按需计算，拒绝浪费**：传统的 `for` 循环 DP 需要开辟巨大的数组并遍历所有可能的状态空间。而 DFS 只会顺着概率的分支去探索，**永远不会访问到无法到达的无效状态**。
2. **无需关心拓扑序**：在复杂的概率图中，状态的转移方向往往是错综复杂的。DFS 利用系统调用栈，自动帮你深入到终点，再把结果**完美地逆向推导（回溯）**回来。
3. **边界极其清晰**：你只需要关心“终点是什么”，在函数开头写几个 `if` 拦截即可，彻底告别了令人头疼的数组初始化。
