# 🏆 CSP 备考与 Debug 终极避坑指南

## 🐛 一、 个人 Debug 血泪史（核心避坑法则）

1. **数据类型与命名警惕**
   - 注意数据类型命名的时候不要混淆了，会导致隐式类型转换和严重的数据丢失。
2. **防范低级手误**
   - 提交前务必**逐行阅读代码**，小心发生手误（如 `i` 写成 `j`，`==` 写成 `=`）。
3. **时间复杂度优化思路**
   - 发现时间复杂度要优化，**第一个考虑二分答案**。注意讨论下二分答案的时候 `judge` 函数返回的含义，**千万不要把左右边界的变化写反了**（死循环的重灾区）。
   - **第二个考虑使用前缀和或者是差分数组**，将 $$O(N)$$ 的查询降为 $$O(1)$$。
4. **函数返回值一致性**
   - 注意函数声明的返回值类型和实际 `return` 的返回值类型必须是一样的！否则可能引发未定义行为。
5. **警惕隐式溢出（十年 OI 一场空，不开 long long 见祖宗）**
   - 注意一个式子定义的数据类型和实际算出来的数据类型不一致时会有数据丢失。
   - **高度警惕 `int * int`**！即使你把结果赋给 `long long`，计算过程中也会先爆 `int`。必须强转：`(ll)a * b` 或 `1ll * a * b`。
6. **差分/前缀和的边界细节**
   - 使用差分数组时，**注意确定边界条件的位置**。
   - 必须看清楚区间**可不可以取等**，对于 `+1` 还是 `-1` 的位置要辨析得清清楚楚（如区间 `[L, R]` 加 $$v$$，差分数组是 `d[L] += v`, `d[R+1] -= v`）。
7. **边界处理的优雅化（拒绝 if-else 迷宫）** ⭐ *NEW*
   - 在处理二维矩阵或区间边界时，如果发现自己为了防止越界写了大量的 `if-else`（比如分情况讨论四个角、四条边），**立刻停手**！这不仅恶心而且极易出错。
   - **防守动作**：使用 `max` 和 `min` 强制将坐标拉回合法范围。例如：`int x1 = max(1, i - r); int x2 = min(n, i + r);`，代码瞬间清爽，彻底告别越界。
8. **比较时的除法转乘法（防范向下取整与精度丢失）** ⭐ *NEW*
   - C++ 中的整数除法是**向下取整**的，在判断平均值是否满足某个阈值时（如 `sum / count <= t`），极易因为取整导致误判！千万不要用 `double`，浮点数有精度误差且运算慢。
   - **防守动作**：永远把不等式两边同时乘以分母，**把除法变成乘法**：`sum <= t * count`。全程整数运算，绝对不丢失精度！

## 🗡️ 二、 CSP T2 核心“时间优化”武器库

CSP 的 T2 本质上是区分“暴力选手”和“进阶选手”的分水岭，核心目标是将 $$O(N^2)$$ 降维到 $$O(N)$$ 或 $$O(N \log N)$$。

1. **前缀和 / 差分数组**
   - **适用场景**：频繁求“区间和”，或者频繁对“区间进行加减”。
2. **二分查找 / 二分答案**
   - **适用场景**：题目具有“单调性”（例如：求“最大值中的最小值”、“满足条件的最少时间/最大距离”）。
3. **排序 + 贪心**
   - **适用场景**：数据乱序，但“排个序之后，按某种规则从左往右拿，结果一定最优”（如：活动安排、区间合并）。
4. **双指针 / 滑动窗口**
   - **适用场景**：在一个数组里找一段连续的区间满足某条件。两个指针一前一后往前扫，绝不回头，时间复杂度 $$O(N)$$。
5. **哈希表（`std::map` / `std::unordered_map`）**
   - **适用场景**：需要快速查找某个数“存不存在”或者“出现了几次”，直接 $$O(\log N)$$ 或 $$O(1)$$ 秒杀。

## 🕳️ 三、 CCF 祖传挖坑手法与防御指南

算法写对了不代表能拿满分，务必在脑海中过一遍这“四大天坑”：

1. **重复数据坑（相同值的处理）**
   - **防守动作**：排序后，如果相邻元素相等（如相同的 $$y$$ 值），要想清楚是直接跳过（`continue`），还是需要合并处理？不处理极易算出虚假的高分。
2. **数据范围坑（爆 `int`）**
   - **防守动作**：看一眼题目给的数据范围。如果 $$N = 10^5$$，每次累加的值也是 $$10^5$$，总和最大会变成 $$10^{10}$$，必定爆 `int`！
   - **口诀**：只要涉及求和、乘积、求最大值，变量无脑开 `long long`。
3. **数组越界坑（爆栈/段错误）**
   - **防守动作**：只要代码里出现了 `a[i-1]`、`a[i+1]`，立刻警觉！必须加上 `if (i == 0)` 或者 `if (i == n-1)` 的特判，防止访问 `-1` 下标。
4. **极端数据坑（$$N=0$$ 或 $$N=1$$）**
   - **防守动作**：假设输入只有 1 个数据，你的代码会不会直接崩溃？循环能不能正常跑完？特判极端情况。

## 🎯 四、 CSP 考场得分战术

- **T1（签到题）**：仔细读题，写基础逻辑，注意别爆 `int`，稳拿 100 分。
- **T2（分水岭）**：
  1. 先写一个 $$O(N^2)$$ 的暴力解法**保底**（拿 60-70 分）。
  2. 看着暴力代码，思考能不能用**二分/前缀和/贪心/map**优化掉一层循环（冲刺 100 分）。
  3. 仔细检查**边界 if** 和特殊情况。
- **T3 / T4（压轴题）**：通常是大模拟、动态规划或复杂图论。**不要死磕正解**，能写多少暴力就写多少暴力，把能拿的部分分（如特殊性质的数据）全部骗到手，拿到 20-40 分就是血赚！
